# 실행 방법 (다음에 이거만 보면 됨)

## TL;DR — 3단계
1. **TradingView 켜기** — PowerShell에서 프로젝트 폴더로 이동 후 런처 실행.
   ```powershell
   cd "D:\01 WORK\260606 stock service"
   powershell -ExecutionPolicy Bypass -File .\start-tradingview.ps1
   ```
   `준비 완료` 또는 `이미 CDP가 연결돼 있습니다`가 뜨면 성공.
2. **Claude에게 확인 요청** — "tv_health_check 해줘" → `cdp_connected: true` 확인.
3. **분석 요청** — "삼성전자 차트 분석해", "에스티팜으로 바꿔줘" 등.

---

## 왜 이게 필요한가 (오늘 막혔던 이유)
- TradingView는 **MSIX(스토어) 패키지**라 일반 경로에 없어서 MCP의 `tv_launch`가 못 찾음. → 런처가 `Get-AppxPackage`로 실행 파일을 동적으로 찾아 해결.
- MCP가 차트를 읽으려면 앱이 **CDP 디버깅 포트(9222)**로 떠 있어야 함. → 런처가 `--remote-debugging-port=9222`로 실행.
- PowerShell 5.1은 **`&&`를 못 씀**. 명령은 `;`로 잇거나 따로 실행.
- 경로의 `~`는 홈 폴더로 해석됨. MCP 서버 실제 위치는 `프로젝트\~\tradingview-mcp-jackson`.

## 이미 끝나 있는 것 (다시 안 해도 됨)
- MCP 서버 `npm install` 완료 (`~\tradingview-mcp-jackson\node_modules` 존재).
- MCP 서버는 Claude Code 시작 시 자동 연결됨. **매번 할 일은 위 1단계(앱 켜기)뿐.**

## 문제 생기면
| 증상 | 해결 |
|------|------|
| `tv_health_check`가 `cdp_connected: false` | 런처(`start-tradingview.ps1`) 다시 실행 |
| 런처가 "설치돼 있지 않습니다" | 프로젝트 폴더의 `TradingView.msix` 먼저 설치 |
| `.ps1` 실행 시 한글 깨짐/파서 오류 | 파일이 UTF-8(BOM) 인지 확인 — 이미 BOM으로 저장돼 있음 |
