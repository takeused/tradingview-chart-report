# 실행 방법 (다음에 이거만 보면 됨)

## TL;DR — 3단계
1. **TradingView 켜기** — PowerShell에서 `.repo` 폴더로 이동 후 런처 실행.
   ```powershell
   cd "D:\01 WORK\260606 stock service\.repo"
   powershell -ExecutionPolicy Bypass -File .\start-tradingview.ps1
   ```
   `준비 완료` 또는 `이미 CDP가 연결돼 있습니다`가 뜨면 성공.
2. **Claude에게 확인 요청** — "tv_health_check 해줘" → `cdp_connected: true` 확인.
3. **분석 요청** — "삼성전자 차트 분석해", "에스티팜으로 바꿔줘" 등.

---

## 왜 이게 필요한가 (막혔던 이유)
- TradingView는 **MSIX(스토어) 패키지**라 일반 경로에 없어서 MCP의 `tv_launch`가 못 찾음. → 런처가 `Get-AppxPackage`로 패키지를 동적으로 찾아 해결.
- MCP가 차트를 읽으려면 앱이 **CDP 디버깅 포트(9222)**로 떠 있어야 함. → 런처가 `--remote-debugging-port=9222` 인자로 실행.
- **MSIX/UWP 앱은 exe를 직접 `Start-Process`하면 활성화되지 않고 즉시 종료됨**(프로세스 0개, 포트 안 열림). → 런처가 Windows `ApplicationActivationManager` COM API로 **AUMID에 디버그 인자를 넘겨** 실행하도록 고쳐짐. (이전의 exe 직접 실행 방식은 이 환경에서 작동 안 함)
- `shell:AppsFolder\<AUMID>` 활성화는 앱은 뜨지만 커맨드라인 인자를 못 넘김 → 그래서 COM API가 필요.
- PowerShell 5.1은 **`&&`를 못 씀**. 명령은 `;`로 잇거나 따로 실행.
- 경로의 `~`는 홈 폴더로 해석됨. MCP 서버 실제 위치는 `프로젝트\~\tradingview-mcp-jackson`.

## 이미 끝나 있는 것 (다시 안 해도 됨)
- MCP 서버 `npm install` 완료 (`~\tradingview-mcp-jackson\node_modules` 존재).
- MCP 서버는 Claude Code 시작 시 자동 연결됨. **매번 할 일은 위 1단계(앱 켜기)뿐.**

## 문제 생기면
| 증상 | 해결 |
|------|------|
| `tv_health_check`가 `cdp_connected: false` | 런처(`start-tradingview.ps1`) 다시 실행 |
| 앱은 뜨는데 포트 9222가 안 열림 | 런처가 COM 활성화 방식인지 확인(아래 참고). exe 직접 실행으로 띄운 인스턴스는 디버그 포트가 없으니 종료 후 런처로 다시 실행 |
| 런처가 "설치돼 있지 않습니다" | 프로젝트 폴더의 `TradingView.msix` 먼저 설치 |
| 런처가 30~45초 대기 후 실패 | 앱이 켜지는 중일 수 있음. 한 번 더 실행하거나, 앱이 완전히 뜬 뒤 `tv_health_check` 재시도 |
| `.ps1` 실행 시 한글 깨짐/파서 오류 | 파일이 UTF-8(BOM) 인지 확인 — 이미 BOM으로 저장돼 있음 |

> 런처 동작 방식: `start-tradingview.ps1`은 기존 인스턴스를 종료한 뒤 `ApplicationActivationManager` COM API로 AUMID(`패키지패밀리명!앱ID`, 매니페스트에서 자동 추출)에 `--remote-debugging-port=9222`를 넘겨 실행한다. AUMID는 현재 `TradingView.Desktop_n534cwy3pjxzj!TradingView.Desktop`.

## 비교 리포트 갱신·푸시 (= Cloudflare 자동 배포)
분석 결과는 `output\stock_comparison_report.html`. 정적 파일은 `public\` 폴더에서 서빙되므로 거기로 복사해 올린다.
```powershell
cd "D:\01 WORK\260606 stock service\.repo"
copy "..\output\stock_comparison_report.html" public\
git add public\stock_comparison_report.html
git commit -m "Update report"
git push
```
- 레포: https://github.com/takeused/tradingview-chart-report
- 로컬 작업 폴더: `D:\01 WORK\260606 stock service\.repo` (원격 연결 완료 상태)
- push하면 Cloudflare가 `npx wrangler deploy`로 자동 배포한다.

## Cloudflare 배포 구성
- `wrangler.toml`이 정적 자산(Workers Assets)을 서빙한다. `[assets] directory = "./public"`.
- 배포할 정적 파일은 모두 `public\` 안에 둔다. 루트의 `.ps1`·`.md`·`.git` 등은 배포에서 제외됨.
- 루트 URL은 `public\index.html`이 `stock_comparison_report.html`로 자동 이동시킨다.
- Cloudflare 대시보드의 deploy command는 `npx wrangler deploy` 그대로 두면 된다.
